WaveCounter™ Liquidity Pool Management System (LPM)
Principles v1.0 (Canonical)

1. Structural Primacy
Liquidity structure is the primary object of control.
Asset identity, price dynamics, and nominal returns are secondary and may not be used as decision drivers without structural validation.

2. Non-Equivalence of Assets
Assets within the pool are not equal.
Each asset must be assigned a functional role:
Anchor


Carrier


Buffer


Speculative


Failure to define roles results in uncontrolled fragility growth.

3. Constrained Liquidity Movement
Liquidity must move only through defined pathways.
Free or unrestricted transitions between assets are prohibited.
Each transition must be:
structurally valid


directionally justified



4. ε-Control Principle
Structural resolution (ε) must be explicitly defined.
Different assets may operate under different ε


Multi-scale ε combinations are allowed


ε must not be implicitly changed during operation


Uncontrolled ε leads to structural distortion.

5. Structural Energy (E) Governance
All decisions must be consistent with the current value of structural energy (E).
E is a derived variable and cannot be replaced by:
price


PnL


external indicators


Actions conflicting with E are invalid.

6. Regime Awareness
The system must operate within clearly identified regimes:
Growth


Stabilization


Protection


Redistribution


Operating outside a defined regime is prohibited.

7. Reserve Requirement
A reserve layer is mandatory.
The system must maintain:
sufficient reserve capacity


defined activation conditions


Operating without reserves is structurally invalid.

8. Fragility Constraint
Growth of fragility must be continuously monitored and constrained.
No action is allowed if it results in:
uncontrolled fragility increase


loss of compensation capacity



9. Irreversibility Rule
Actions must not lead the system into irreversible states.
If reversibility cannot be guaranteed:
action is prohibited

10. No-Action Validity
The absence of action is a valid and often optimal system response.
Forced activity is a violation of system integrity.

11. Yield as Emergent Property
Yield must not be pursued directly.
It is an emergent result of:
structural stability


controlled redistribution


Direct yield targeting leads to fragility accumulation.

12. Execution Independence
System validity must not depend on execution method.
The system must remain consistent under:
manual execution


semi-automated execution


fully automated execution



13. Asset-Agnostic Applicability
The system must remain valid regardless of:
asset class


market type


pool size


Any dependency on specific assets is a structural flaw.

14. WLRT Consistency
All system decisions must remain consistent with:
Wave Liquidity Redistribution Theory
Any contradiction invalidates the system state.

15. Structural Integrity Over Performance
System integrity has absolute priority over performance.
Any performance improvement achieved through structural violation is invalid.

Canonical Statement
A liquidity pool is not managed through prediction or allocation, but through structural control, where stability defines the boundaries of all valid actions.
